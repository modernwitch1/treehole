# 浙工商树洞美食模块：商家员工使用手册

版本：2026-07

## 1. 你使用的是什么系统

商家后台地址：`https://merchant.unidating.top`

这是食堂商家专用运营后台，不是校园论坛。你只能看到被授权商家的窗口、产品、宣传内容和学生匿名评价，不能看到论坛帖子、私信、聊天室、其他商家或学生真实身份。

如果你是平台超级管理员，可在登录框使用论坛用户名或论坛邮箱及同一密码；普通员工必须使用平台管理员发放邀请后创建的商家后台账号。

## 2. 第一次使用：接受邀请

1. 从平台管理员收到个人邀请链接。
2. 在有效期内打开链接；邀请一般有效 72 小时。
3. 填写后台显示名称和至少 8 位密码。
4. 阅读并勾选商家后台服务条款。
5. 提交后系统会自动登录并进入被授权店铺。

邀请链接是一次性的，不能转发。若链接过期、显示“已使用”或提示商家不可用，请联系平台管理员重新发放；不要反复猜测 token。

## 3. 角色说明

| 角色        | 权限                                                      |
| ----------- | --------------------------------------------------------- |
| 店主 owner  | 修改店铺资料；编辑窗口；新增/编辑产品和宣传内容；回复评价 |
| 编辑 editor | 编辑窗口；新增/编辑产品和宣传内容；回复评价               |
| 只读 viewer | 查看产品、宣传内容和评价，不能提交修改                    |

按钮是否显示由角色决定，服务端也会再次校验。看到“没有权限”时不要尝试绕过页面或调用接口。

## 4. 店铺首页

首页顶部显示当前店铺、账号和数据概览。如果一个账号被授权多个店铺，可用右上角下拉框切换。平台管理员看到的是全部已启用商家。

页面卡片含义：

- 产品：当前店铺的产品记录，包括审核状态和在售状态；
- 宣传内容：新品、促销、广告、公告；
- 用户评价：对应窗口的匿名评价和意见反馈。

学生身份在后台统一显示为“匿名同学”。不要根据内容、时间或订单信息猜测发言人，也不要在回复中索要姓名、学号、手机号。

## 5. 店铺资料

店主可以维护店铺名称、简介和对外联系方式。保存前检查：

- 名称与线下招牌一致；
- 联系方式只填写愿意公开给全校学生的内容；
- 简介不要放个人手机号、内部账号、收款密码或学生信息。

编辑员工可以看到资料，但不能保存。店铺状态由平台管理员控制。

## 6. 窗口管理

“经营窗口”显示食堂、楼层、窗口编号和位置说明。店主和编辑可以修改窗口名称、编号和位置，也可以暂时停用/重新启用窗口。

停用窗口的影响：

- 学生端不再展示该窗口的可售产品和新内容；
- 历史评价不会被删除；
- 不能向停用窗口创建新的产品或宣传内容；
- 重新启用由店主/编辑操作后立即生效，但商家本身和食堂仍必须可用。

窗口名称、编号修改前先确认与线下实际位置一致，避免学生找错窗口。

## 7. 产品管理

### 7.1 新增产品

在“上架产品”填写产品名、价格、所属窗口和产品介绍，提交后进入平台审核。价格以元填写，系统按分保存；没有固定价格可以留空，但建议写清规格和计价方式。

产品状态：

- `审核中`：等待平台审核；
- `已上线`：通过审核；
- `已下架`：被平台隐藏或下架；
- `草稿/已删除`：内部状态，按页面提示处理。

### 7.2 编辑和停售

点击产品“编辑”可以修改名称、介绍、价格和窗口。名称或介绍改变后会显示“保存并重新审核”，旧内容不会直接覆盖学生端；管理员重新通过后才公开。

“暂停售卖/恢复售卖”只改变即时在售开关，不改写产品文案。临时缺货、窗口换班时优先使用停售，不要删除产品。

发布前检查：产品名、价格、规格、窗口、过敏原/特殊说明是否准确。不要发布与实际不符的“全校最低价”等无法核实承诺。

## 8. 新品、促销和广告

在“发布新品 / 广告”选择类型，填写标题、所属窗口和正文后提交。内容必须经过平台审核；提交成功不等于已经对学生公开。

可发布的类型：新品、促销、广告、公告。建议正文包含：

- 活动或产品的具体内容；
- 开始/结束时间、适用窗口和限制条件；
- 学生需要知道的价格或规格变化；
- 必要的食品安全和过敏原提示。

在“发布记录”中可以编辑已提交内容。标题或正文改变后会重新进入审核；编辑窗口归属、类型等信息也应在提交前仔细复核。已定时内容需要检查发布时间和失效时间的先后关系。

不要发布：

- 学生个人信息、订单截图或联系方式；
- 外部收款二维码、私人转账要求或与平台无关的链接；
- 夸大、侮辱竞争商家、攻击学生的内容；
- 未经确认的食品安全、医疗或功效承诺。

## 9. 查看评价和回复意见

评价按窗口显示，包括口味评分、文字评价和意见反馈。回复由平台审核后才会公开。

建议回复结构：

1. 先确认已看到意见；
2. 对能改进的事项给出具体行动或时间；
3. 对无法立即解决的事项说明原因和可行渠道；
4. 不争吵、不猜测身份、不要求学生公开个人资料。

回复提交后会显示审核状态。若回复失败，页面会显示错误信息；先保留草稿，确认网络恢复后再提交，避免重复发送。

## 10. 登录安全与会话

- 使用 HTTPS 地址登录，不要在公共电脑保存密码；
- 不与同事共用账号；应由管理员给每位员工单独邀请；
- 离岗时点击“退出”；
- 如果页面提示登录过期，系统会自动尝试刷新安全会话；仍失败时重新登录；
- 怀疑密码泄露、账号被他人使用或员工离职，立即联系平台管理员撤销账号；
- 管理员不会向你索要密码。

## 11. 常见问题

### 为什么学生看不到我刚提交的产品？

新产品、修改后的产品和宣传内容都要通过平台审核；同时检查商家、食堂、窗口是否启用，以及产品是否处于“在售”。

### 为什么不能选择某个窗口？

停用窗口不能接收新产品/内容。请让店主或平台管理员重新启用，或选择当前运营窗口。

### 为什么编辑后又变成审核中？

这是正常的版本保护机制。涉及学生可见文案的修改必须重新审核，避免已通过内容被直接替换。

### 我能看到学生是谁吗？

不能。商家后台只提供匿名展示，这是平台保护学生和商家的基本边界。任何试图通过评价内容推断身份并公开的行为都不被允许。

### 账号没有店铺权限怎么办？

把错误提示和登录邮箱告诉平台管理员，让管理员检查邀请、商家状态和员工关系。不要注册论坛账号来替代商家后台账号。

## 12. 每日运营清单

- 开档前确认窗口状态和产品在售状态；
- 新品、促销和广告至少提前留出审核时间；
- 价格、活动期限和适用窗口变更后及时编辑并重新提交；
- 每天查看新评价，优先回复食品安全、过敏原和服务改进意见；
- 收档或缺货时及时停售，恢复供应后再恢复售卖；
- 离职/调岗立即通知平台管理员撤销或调整权限。
