PART 0你必须知道的概念（5 分钟）

如果你第一次接触服务器，先花 5 分钟搞清楚这几个词。知道了之后后面的步骤就不会懵了。

词汇白话解释类比
EC2亚马逊租给你的一台「真实电脑」，在香港的机房里 24 小时开着你租的网吧电脑，但从不关机
SSH一种「远程控制」技术，让你在自己的 Mac 里打命令，命令在服务器上执行远程桌面，但只有命令行
.pem 文件你的「钥匙」，证明你有权进入这台服务器。只有一份，丢了就进不去了门禁卡
安全组服务器的「防火墙规则」，控制哪些端口对外开放公寓的门卫
弹性 IP一个固定不变的公网 IP 地址。不绑的话每次重启 IP 会变固定电话号码
Caddy反向代理服务器，负责把用户的请求转发给你的 Node 程序，并自动申请 HTTPS 证书前台接待员
PM2Node.js 进程守护，程序崩溃了自动重启，服务器重启后自动拉起保姆
Prisma 迁移把 schema.prisma 里定义的表结构「应用」到真实数据库里给数据库「装修」
整体架构（部署完成后长这样）： 用户浏览器 │ HTTPS 443 ▼ Caddy（反向代理） ┌─────────────────────────────────────────────────────┐ │ forum.你的域名.hk → PM2 运行的 Next.js Web :3001 │ │ api.你的域名.hk → PM2 运行的 NestJS API :3000 │ │ admin.你的域名.hk → PM2 运行的 Next.js Admin :3002 │ └─────────────────────────────────────────────────────┘ │ │ PostgreSQL :5432 Redis :6379 (同一台 EC2， (同一台 EC2， 仅监听 localhost) 仅监听 127.0.0.1) │ AWS SES（发验证邮件） AWS S3（存图片文件）
💡
读这篇教程时，你需要准备：

① Mac 电脑（Windows 也行，SSH 命令一样） ② 一张可以在国际网站消费的信用卡（注册 AWS 用，不会扣费） ③ 你的 HK 域名（如 zjgsu.hk）的管理权

PART 1准备工作：注册 AWS & 切换香港区

1.1 注册 AWS 账号

  1. 打开 aws.amazon.com，点右上角「创建 AWS 账户」。

    填写邮箱、密码、账号名（随意）。邮箱最好用自己常用的。

  2. 选「个人账户」，填写姓名、地址（可以填香港地址）。

  3. 输入信用卡信息，AWS 会扣 $1 验证，随后退回。不会真正收费，除非你超过免费额度或使用付费服务。

  4. 手机验证通过后，选择「基本支持计划（免费）」。

  5. 注册成功！登录到 AWS 控制台。

🚨 可能出错
症状

信用卡验证失败 / 一直转圈

原因

国内银行卡有外币限制，或卡里余额不足（需要约 ¥7）

解决

换一张开通了「外币支付」的卡；或用 Visa/Mastercard 外币卡。支付宝 Visa 国际版一般可以。

1.2 切换到香港区

AWS 控制台默认显示「美国东部」区域。我们要改成香港。

  1. 登录后，看右上角，有个区域名称（比如「us-east-1」或「亚太区域（首尔）」）。

  2. 点击它，在下拉菜单里找「亚太地区（香港）ap-east-1」并点击。

🚨 可能出错
症状

下拉菜单里找不到香港区

原因

香港区是「选择性加入」区域，需要手动启用

解决

进入「账户设置」→「AWS 区域」→ 找到「亚太地区（香港）」→ 点「启用」→ 等几分钟刷新页面。路径：控制台右上角头像 → 账单和成本管理 → 账户 → 滚到「AWS 区域」一栏。

PART 2创建 EC2 实例

⚠️
确认你现在在「亚太（香港）ap-east-1」区

右上角必须显示「亚太地区（香港）」。在错误的区域创建实例，流量费会更贵而且延迟高。

2.1 启动实例

  1. 在 AWS 控制台顶部搜索框输入「EC2」，点击进入 EC2 控制台。

  2. 点击橙色按钮「启动实例」。

  3. 名称：随意，比如 zjgsu-forum。

  4. 操作系统镜像（AMI）：选「Ubuntu Server 24.04 LTS」，架构选「64 位 (x86)」。

    💡

    为什么选 Ubuntu 24.04？因为它有官方 PostgreSQL 16 和 Redis 7 的源，长期支持（LTS）到 2029 年，社区文档最多。

  5. 实例类型：选 t3.small（2 vCPU，2 GB 内存）。

    实例内存建议用途香港月费（约）
    t3.micro1 GB❌ 不够用，编译 Next.js 会 OOM~$8
    t3.small2 GB✅ 推荐起步~$16
    t3.medium4 GB流量大了再升级~$32

  6. 密钥对（登录）：点「创建新密钥对」。

    • 名称：zjgsu-forum-key
    • 密钥对类型：RSA
    • 私有密钥文件格式：.pem（Mac/Linux 用这个）
    • 点「创建密钥对」，浏览器自动下载 hezhong666.pem
    🔐
    这个 .pem 文件非常重要！

    AWS 只让你下载一次。丢了就只能销毁实例重建。立刻把它移动到安全的地方，比如 ~/.ssh/hezhong666.pem，并备份到 iCloud 或加密压缩包。

  7. 存储：把根卷从默认的 8 GB 改成 30 GB。依赖 + 编译产物 + 数据库加起来很容易超 8 GB。

  8. 网络设置：先不动，等到「安全组」一节再改。

  9. 点右侧「启动实例」。等 1-2 分钟，状态变为「正在运行」即可。

🚨 可能出错：实例一直处于「待定」状态
原因

香港区资源偶尔不足，或账号刚注册需要等待验证。

解决

等 5 分钟刷新。如果超过 10 分钟还是「待定」，尝试换可用区（AZ）重建，或提交 AWS Support。

2.2 配置安全组（防火墙）

默认安全组可能只开了 22 端口，我们需要再加 80 和 443。

  1. 在 EC2 控制台，点「实例」，找到刚创建的实例，点击它的 ID 进入详情。

  2. 点「安全」选项卡 → 点安全组的名字 → 点「编辑入站规则」。

  3. 确保有以下规则（没有的就「添加规则」）：

    类型端口来源说明
    SSH22我的 IP（My IP）只允许你的 IP 登录，更安全
    HTTP80任何地方（0.0.0.0/0）HTTP 访问
    HTTPS443任何地方（0.0.0.0/0）HTTPS 访问

  4. 点「保存规则」。

🚫
千万不要开放 5432（PostgreSQL）或 6379（Redis）端口给外网！

数据库只应该在服务器内部访问。一旦开放，全球扫描机器人会在几分钟内找到并暴力破解你的数据库。

🚨 可能出错：设置了「My IP」但后来 SSH 连不上了
原因

家用宽带 IP 是动态的，换了网络或重拨就变了

解决

回到安全组，把 SSH 的来源改成你的新 IP（Google 搜「我的 IP」就能看到），或者暂时改成 0.0.0.0/0，登录进去做完配置后再改回来。

2.3 绑定弹性 IP（固定公网 IP）

不绑弹性 IP 的话，每次重启实例公网 IP 会变，域名解析就失效了。

  1. EC2 控制台左侧菜单 → 「网络与安全」→「弹性 IP」→ 点「分配弹性 IP 地址」→ 点「分配」。

  2. 选中刚分配的 IP → 「操作」→「关联弹性 IP 地址」。

    选择你的实例，实例的私有 IP 也选一下，点「关联」。

  3. 回到「实例」页，你的实例现在会显示一个固定的「弹性 IP 地址」，记下来，后面配置 DNS 要用。

⚠️
弹性 IP 不用时会收费

弹性 IP 分配了但没有关联到运行中的实例，或实例被停止了，AWS 会收约 $0.005/小时 的费用。所以要么绑着用，要么释放掉。

PART 3第一次 SSH 连接服务器

SSH 就是「远程控制命令行」。你在自己的 Mac 终端里打命令，命令在服务器上执行。

3.1 设置密钥文件权限

先在自己的 Mac 上操作（打开「终端」应用，不需要移动 .pem 文件，放在下载目录就行）：

# 设置正确权限（SSH 强制要求，权限太宽会被拒绝）
chmod 400 ~/Downloads/hezhong666.pem

# 确认一下
ls -la ~/Downloads/hezhong666.pem
# 应该看到类似：-r--------  1 你的用户名 ...
💡
为什么要 chmod 400？

SSH 出于安全原因，发现密钥文件权限太开放（别人也能读）就直接拒绝使用。chmod 400 表示「只有你自己可以读，其他人什么权限都没有」。

🚨 可能出错：chmod 报错 No such file or directory
原因

.pem 文件不在 ~/Downloads，或者文件名不一样

解决

在 Finder 里找到 .pem 文件，右键 → 「在终端中打开所在文件夹」，然后运行 chmod 400 hezhong666.pem。SSH 命令里的 -i 路径也要对应修改。

3.2 第一次登录

# 把 你的弹性IP 替换成刚才记下来的那个 IP 地址
ssh -i ~/Downloads/hezhong666.pem ubuntu@你的弹性IP

# 例如：
ssh -i ~/Downloads/hezhong666.pem ubuntu@43.198.xxx.xxx

第一次连接会问你「确认连接吗？」，输入 yes 回车。连上后终端提示符会变成 ubuntu@ip-xxx-xxx: 开头，说明你现在在服务器上了！

🚨 可能出错：ssh: connect to host xxx port 22: Connection timed out
原因

安全组 22 端口没开，或者你的 IP 不在白名单里

解决

回到 AWS 控制台 → 安全组 → 入站规则，确认 22 端口的来源包含你当前的 IP。可以临时改成 0.0.0.0/0 测试。

🚨 可能出错：WARNING: UNPROTECTED PRIVATE KEY FILE!
原因

.pem 文件权限太宽（比如 644），SSH 拒绝使用

解决

运行 chmod 400 ~/Downloads/hezhong666.pem 再重试。

🚨 可能出错：Permission denied (publickey)
原因

用了错误的密钥文件，或者用了错误的用户名（Ubuntu 系统默认用户是 ubuntu，不是 root 或 ec2-user）

解决

确认命令里是 ubuntu@ 开头；确认 -i 后面指向的是正确的 .pem 文件。

🚨 可能出错：Host key verification failed
原因

这个 IP 之前绑定过另一台服务器，SSH 认为「换人了」，拒绝连接

解决

运行 ssh-keygen -R 你的IP，清除旧记录，再重新连接。

成功登录后，先做两件事：

# 更新系统软件包列表（必做）
sudo apt update

# 如果弹出交互菜单问「restart services」，按回车选默认 OK 即可
🚨 可能出错：apt update 报错 Could not get lock /var/lib/dpkg/lock
原因

Ubuntu 启动后会自动跑一次系统更新，这个进程还没结束，锁住了 apt

解决

等 2-3 分钟再试；或者运行 sudo systemctl stop unattended-upgrades 等它停下，再 sudo apt update。

PART 4安装运行环境

接下来所有命令都在服务器上执行（SSH 连接后的窗口）。

4.1 先加 Swap 内存（重要！别跳过）

t3.small 只有 2 GB 内存。编译 Next.js 时内存不够会被系统直接 kill 掉。加 2 GB Swap（用硬盘当内存），保护我们。

# 检查当前是否有 swap（没有输出就是没有）
swapon --show

# 创建 2GB swap 文件
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 让 swap 重启后自动挂载
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 验证
free -h
# 应该能看到 Swap: 2.0Gi
🚨 可能出错：fallocate failed: Operation not supported
原因

某些文件系统（btrfs）不支持 fallocate 创建 swap

解决

换用 dd 命令：sudo dd if=/dev/zero of=/swapfile bs=1M count=2048，然后继续 chmod 600 后面的步骤。

4.2 安装 Node.js 22（通过 nvm）

nvm 是「Node 版本管理器」，可以在同一台机器上装多个版本的 Node 并随时切换。

# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# ⚠️ 重要：让 nvm 命令生效（不重新 SSH，就手动 source）
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# 安装 Node.js 22
nvm install 22

# 设为默认版本
nvm alias default 22

# 验证
node -v   # 应该显示 v22.x.x
npm -v    # 应该显示 10.x.x
🚨 可能出错：nvm: command not found（安装后立刻运行就报这个）
原因

nvm 安装脚本修改了 ~/.bashrc，但当前 shell 还没有读取

解决

运行 source ~/.bashrc，或者断开 SSH 重新连接，然后再运行 nvm install 22。

🚨 可能出错：curl 下载超时 / 连接失败
原因

香港 EC2 访问 GitHub raw 内容偶尔不稳定

解决

等几分钟重试，或者用 wget 替代：wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

4.3 安装 pnpm 9

# 用 npm 全局安装 pnpm
npm install -g pnpm@9

# 验证
pnpm -v   # 应该显示 9.x.x
🚨 可能出错：pnpm: command not found（安装后找不到）
原因

npm 全局安装的位置不在 PATH 里

解决

运行 npm bin -g 查看全局 bin 目录，然后把它加到 PATH：export PATH="$(npm bin -g):$PATH"。或者直接重新打开终端（SSH 断开再连）。

4.4 安装 PostgreSQL 16

Ubuntu 24.04 自带的 apt 源里是 PostgreSQL 16，直接安装即可（无需手动添加 pgdg 源）。

# 安装 PostgreSQL 16
sudo apt install -y postgresql postgresql-contrib

# 启动并设置开机自启
sudo systemctl enable postgresql
sudo systemctl start postgresql

# 验证是否在运行
sudo systemctl status postgresql
# 看到 Active: active (running) 就对了

# 再验证版本
psql --version
# 应该显示 psql (PostgreSQL) 16.x
🚨 可能出错：postgresql.service 启动失败
诊断

运行 sudo journalctl -xeu postgresql 看详细错误日志。

常见原因：端口 5432 被占用

运行 sudo lsof -i :5432 查看是什么占用，用 sudo kill -9 <PID> 杀掉，再重启。

常见原因：磁盘满了

运行 df -h，如果 / 是 100%，删除一些文件或扩容 EBS 卷。

4.5 安装 Redis 7

# Ubuntu 24.04 默认源已经有 Redis 7
sudo apt install -y redis-server

# 设置开机自启 + 启动
sudo systemctl enable redis-server
sudo systemctl start redis-server

# 验证
redis-cli ping
# 应该返回 PONG
🚨 可能出错：NOAUTH Authentication required（之后连接 Redis 报这个）
原因

你给 Redis 设置了密码，但连接时没有传密码

解决

用 redis-cli -a 你设置的密码，或者在 .env 里把 REDIS_URL=redis://:密码@127.0.0.1:6379。

4.6 安装 Caddy（反向代理 + 自动 HTTPS）

Caddy 是一个现代反向代理，比 Nginx 简单很多。最大优点：一行配置就能自动申请并续期 Let's Encrypt 证书，不需要手动跑 certbot。

# 安装依赖
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl

# 添加 Caddy 官方 apt 源
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list

# 安装
sudo apt update
sudo apt install -y caddy

# 验证
caddy version    # 应该显示 v2.x.x
sudo systemctl status caddy    # Active: active (running)
🚨 可能出错：caddy 启动后 80 端口被占用
诊断

运行 sudo lsof -i :80，看是什么在占用 80 端口。

常见原因：apache2 默认安装了

运行 sudo systemctl stop apache2 && sudo systemctl disable apache2，然后重启 Caddy：sudo systemctl restart caddy。

✅ Part 4 安装完成校验 运行以下命令，全部应该有输出：
node -v → v22.x.x
pnpm -v → 9.x.x
psql --version → 16.x
redis-cli ping → PONG
caddy version → v2.x.x

PART 5数据库初始化

PostgreSQL 安装后有一个默认的 postgres 超级用户，但我们要为项目创建一个单独的用户和数据库，更安全。

# 切换到 postgres 系统用户，进入 psql 命令行
sudo -u postgres psql

# 以下命令在 psql 里面执行（提示符是 postgres=#）

-- 创建数据库用户（把 强密码请换成你自己的 改掉！）
CREATE USER forum WITH PASSWORD '强密码请换成你自己的';

-- 创建数据库，指定 owner 是 forum 用户
CREATE DATABASE forum OWNER forum ENCODING 'UTF8' LC_COLLATE 'C' LC_CTYPE 'C' TEMPLATE template0;

-- 给用户授权
GRANT ALL PRIVILEGES ON DATABASE forum TO forum;

-- 验证
\l
-- 应该能看到名为 forum 的数据库，Owner 是 forum

-- 退出 psql
\q
🚨 可能出错：could not connect to server: No such file or directory（psql 连不上）
原因

PostgreSQL 服务没启动

解决

运行 sudo systemctl start postgresql，然后再试。

🚨 可能出错：ERROR: database "forum" already exists
原因

之前创建过同名数据库

解决

如果是全新部署，先 DROP DATABASE forum;，再重建。如果是迁移，不用重建，直接用现有数据库。

接下来给 Redis 加密码（防止服务器上其他进程随意访问）：

# 编辑 Redis 配置文件
sudo nano /etc/redis/redis.conf

# 找到这一行（Ctrl+W 搜索 requirepass）：
# requirepass foobared
# 取消注释并改成你的密码：
requirepass 你的Redis密码

# 确认 bind 只监听本地（找到 bind 那行，应该是）：
bind 127.0.0.1 -::1

# 保存：Ctrl+O 回车，退出：Ctrl+X

# 重启 Redis
sudo systemctl restart redis-server

# 验证（用密码连接）
redis-cli -a 你的Redis密码 ping
# 返回 PONG 就对了

PART 6部署项目代码

6.1 把代码传到服务器

有两种方式：用 Git（推荐）或者 scp（直接传文件夹）。

方式 A：Git（推荐，代码更新方便）

PART 7PM2 进程守护

PM2 负责：① 让 Node 程序在后台运行（不是挂在终端上） ② 程序崩溃自动重启 ③ 服务器重启后自动拉起所有服务。

# 全局安装 PM2
npm install -g pm2

# 验证
pm2 -v

在项目根目录创建 PM2 配置文件：

# 在服务器上执行
cat > /var/www/forum/ecosystem.config.js << 'EOF'
module.exports = {
  apps: [
    {
      name: 'forum-api',
      cwd: '/var/www/forum/apps/api',
      script: 'node',
      args: 'dist/main.js',
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '512M',
      env: {
        NODE_ENV: 'production',
      },
    },
    {
      name: 'forum-web',
      cwd: '/var/www/forum/apps/web',
      script: 'node',
      args: 'node_modules/.bin/next start -p 3001',
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '400M',
      env: {
        NODE_ENV: 'production',
        PORT: '3001',
      },
    },
    {
      name: 'forum-admin',
      cwd: '/var/www/forum/apps/admin',
      script: 'node',
      args: 'node_modules/.bin/next start -p 3002',
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '300M',
      env: {
        NODE_ENV: 'production',
        PORT: '3002',
      },
    },
  ],
};
EOF
# 启动所有服务
cd /var/www/forum
pm2 start ecosystem.config.js

# 查看状态（所有 status 应该是 online）
pm2 status

# ⚠️ 关键步骤：设置开机自启
# 不执行这步，服务器重启后服务就全没了！
pm2 startup
# 这条命令会输出一行 sudo ... 的命令，复制下来执行它！
# 例如：sudo env PATH=$PATH:/home/ubuntu/.nvm/... pm2 startup systemd -u ubuntu --hp /home/ubuntu

# 保存当前进程列表（以后重启自动还原）
pm2 save
⚠️
pm2 startup 一定要执行 pm2 save！

startup 只是注册开机自启的机制，save 才是保存当前进程列表。没有 save，重启后 PM2 不知道要启动哪些服务。

诊断

常见原因：dist/main.js 不存在

常见原因：端口被占用

PART 8Caddy 反向代理 + 自动 HTTPS

💡
做这一步之前，先把域名解析好

在你的域名注册商（阿里云、腾讯云、Cloudflare 等）添加以下 A 记录，指向你的弹性 IP：

记录类型主机记录（子域名）记录值（弹性 IP）TTL
Aforum（或 @）你的弹性IP5分钟
Aapi你的弹性IP5分钟
Aadmin你的弹性IP5分钟

验证 DNS 是否生效：

# 在服务器上运行
dig +short api.你的域名.hk
# 应该返回你的弹性 IP

# 如果 dig 命令不存在：
sudo apt install -y dnsutils
nslookup api.你的域名.hk

DNS 生效（通常 5-10 分钟，最长 48 小时）后，配置 Caddyfile：

# 备份默认配置
sudo cp /etc/caddy/Caddyfile /etc/caddy/Caddyfile.bak

# 写入新配置（把 你的域名.hk 替换成你的真实域名）
sudo tee /etc/caddy/Caddyfile > /dev/null << 'EOF'
# 把 api 子域名的流量转发给 NestJS API（3000 端口）
api.你的域名.hk {
    reverse_proxy localhost:3000
    encode gzip
}

# 把主域名的流量转发给 Web 前端（3001 端口）
forum.你的域名.hk {
    reverse_proxy localhost:3001
    encode gzip
}

# 把 admin 子域名的流量转发给管理后台（3002 端口）
admin.你的域名.hk {
    reverse_proxy localhost:3002
    encode gzip
}
EOF

# 检查配置语法是否正确
caddy validate --config /etc/caddy/Caddyfile

# 重新加载 Caddy（不中断服务）
sudo systemctl reload caddy

# 查看 Caddy 日志
sudo journalctl -u caddy -f

Caddy 会自动向 Let's Encrypt 申请 HTTPS 证书，无需任何额外操作。

🚨 可能出错：Caddy 日志显示 certificate must be renewed（或 TLS handshake error）
原因

DNS 还没有生效，Caddy 申请证书时域名解析不到这台服务器

解决

① 用 dig +short 你的域名 确认 DNS 已经指向正确 IP ② DNS 生效后，Caddy 会自动重试申请证书，通常几分钟内自动解决。

🚨 可能出错：caddy validate 报 unrecognized directive
原因

Caddyfile 语法错误，或者复制时引号变成了中文引号

解决

仔细检查 Caddyfile，确保所有引号和括号都是英文半角字符。用 nano /etc/caddy/Caddyfile 检查。

🚨 可能出错：浏览器提示「ERR_TOO_MANY_REDIRECTS」
原因

前端（Next.js）或 API 里有强制 HTTP→HTTPS 的重定向，和 Caddy 的重定向冲突了

解决

检查 Next.js 的 next.config.ts 里是否有 redirect 配置；确保 NestJS 里没有手动重定向到 HTTPS；Caddy 本身会自动处理 HTTPS，不要再叠加一层。

✅ Caddy 配置成功验证 打开浏览器访问 https://api.你的域名.hk/health，
应该看到 API 返回的 JSON（{"status":"ok"} 类似内容），且浏览器地址栏有🔒绿锁。

PART 9AWS SES 邮件发送配置

SES（Simple Email Service）用于发送注册验证码邮件。

  1. 在 AWS 控制台确认你还在香港区（ap-east-1）。搜索「SES」→ 进入。

  2. 验证发件人邮箱或域名：左侧菜单「已验证的身份」→「创建身份」。

    • 选「电子邮件地址」，填入发件邮箱（如 noreply@你的域名.hk），发邮件到这个地址，点击验证链接。
    • 或者选「域」，验证整个域名（需要在 DNS 添加 TXT 记录）。域验证后可以用这个域名下的任意邮箱发信。

  3. 创建 IAM 用户（程序用的密钥）：在 AWS 控制台搜索「IAM」，进入。

    「用户」→「创建用户」→ 名称：forum-api-user →「仅具有编程访问权限」。

    权限：「直接附加策略」→ 搜索 AmazonSESFullAccess 和 AmazonS3FullAccess，勾选两个（后者用于图片上传）。

    完成后显示 Access key ID 和 Secret access key，立刻复制保存，关闭后不能再查看。

  4. 把这两个值填入 apps/api/.env：

    AWS_ACCESS_KEY_ID=AKIA...你的key
AWS_SECRET_ACCESS_KEY=...你的secret
AWS_REGION=ap-east-1
SES_FROM_EMAIL=noreply@你的域名.hk

# SMTP 配置（使用 SES SMTP 方式）
SMTP_HOST=email-smtp.ap-east-1.amazonaws.com
SMTP_PORT=587
SMTP_USER=# SES SMTP 用户名（在 SES 控制台生成，不是 IAM key！）
SMTP_PASS=# SES SMTP 密码

  5. 重启 API：pm2 restart forum-api

🚨 可能出错：发送邮件报 MessageRejected: Email address is not verified
原因

SES 默认在「沙箱模式」，只能发送给已验证的邮箱地址。这意味着你只能测试，不能给真实用户发。

解决

申请退出沙箱（生产访问）：SES 控制台 → 「账户控制面板」→「申请生产访问权限」，填写用途说明（校园论坛，发注册验证码，预计发送量）。通常 1-3 个工作日审批通过。

🚨 可能出错：SES SMTP 连接超时 / 连不上 port 587
原因

EC2 出站流量默认封锁了 25 端口，但 587 应该是通的。如果 587 也不通，可能是 AWS 账号层面的限制。

解决

① 换用 465 端口（SSL）② 或者在 AWS Support 申请解开 25 端口限制 ③ 测试：telnet email-smtp.ap-east-1.amazonaws.com 587，能连上就显示 「Connected」。

*/30 * * * * df -h / | awk 'NR==2 {gsub(/%/,""); if ($5 > 85) system("echo \"Disk usage: "$5"%\" | mail -s \"[警告] 服务器磁盘快满了\" 你的邮箱@qq.com")}'

# PM2 图形化监控（Ctrl+C 退出）
pm2 monit

# 清理 PM2 日志（防止日志撑满磁盘）
pm2 install pm2-logrotate           # 装日志自动轮转插件
pm2 set pm2-logrotate:max_size 50M  # 单个日志最大 50MB
pm2 set pm2-logrotate:retain 5      # 保留最近 5 个日志文件

12.4 SSH 安全加固（可选但推荐）

# 禁用密码登录（只允许密钥登录，防暴力破解）
sudo nano /etc/ssh/sshd_config

# 确认/修改这两行：
PasswordAuthentication no
PubkeyAuthentication yes

# 重启 SSH 服务（不会断开当前连接）
sudo systemctl restart sshd

# 安装 fail2ban（自动封禁多次失败登录的 IP）
sudo apt install -y fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

部署完成！

整个流程走完，你现在有：一台香港 EC2 服务器（固定 IP、自动 HTTPS）、PostgreSQL 16 数据库、Redis 7 缓存、PM2 守护的三个 Node 服务（API + Web + Admin），以及通过 Caddy 代理的三个子域名。
