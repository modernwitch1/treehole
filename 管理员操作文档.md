# 管理员后台操作文档

## 1. 登录与角色

后台使用管理员账号登录。登录后，系统会根据账号角色自动显示可用功能；前端菜单隐藏不是唯一的权限控制，后端接口也会再次校验权限。

| 角色 | 可用范围 |
| --- | --- |
| 普通管理员 / 版主 | 仅可处理**帖子举报**。只能进入“举报队列”，不能查看用户资料、全站内容、评论、聊天室、审核案件、图片、系统配置或审计日志。 |
| 超级管理员 | 可使用全部后台功能、查看全站信息并进行身份溯源。 |

> 超级管理员权限用于平台治理、校方依法协查和安全事件处置。身份调阅仍会留下审计记录，便于事后核对操作人、目标和时间。

## 2. 普通管理员：举报队列

普通管理员登录后只会看到“举报队列”。队列只返回举报目标为**帖子**的记录；评论、用户、私信、聊天室等其他举报不会暴露给普通管理员。

### 2.1 查看与筛选

- 可按处理状态查看：待处理、已处理、已驳回。
- 可按举报分类筛选：违法/政治、色情低俗、广告/营销、人身攻击、其他。
- 分类筛选在服务端执行，分页总数与筛选结果一致。
- 每条举报含举报时固化的内容快照，原帖之后被删除或修改时仍可作为处理依据。
- 举报人身份对普通管理员脱敏显示。

### 2.2 处理操作

打开一条待处理帖子举报后，可执行：

| 操作 | 结果 |
| --- | --- |
| 隐藏内容 | 将被举报帖子隐藏，并合并处理同一帖的其他待处理举报。 |
| 处理完成 | 记录已处理，不改变帖子可见状态。 |
| 驳回 | 标记为驳回，不改变帖子可见状态。 |

所有处理操作都会记录审计日志。普通管理员不能通过接口处理非帖子举报，也不能直接浏览或处理任意帖子。

## 3. 超级管理员功能

超级管理员拥有以下全部权限。

### 3.1 仪表盘

- 查看用户、帖子、评论、举报、待审核内容等全站统计。
- 查看趋势与待办数量，作为处理优先级参考。

### 3.2 用户与角色

- 搜索、筛选用户；用户列表显示 UID、邮箱、角色、状态、发帖/评论/举报计数及登录信息。
- 对用户执行禁言、封禁、解封、升为版主、降为普通用户等操作。
- 超级管理员账号受保护，不能在用户列表中被降权或处罚。

### 3.3 帖子与评论管理

- 搜索帖子、评论，按状态、板块、是否被举报等条件筛选。
- 隐藏、恢复、置顶、取消置顶、锁定、解锁或删除帖子；处理评论及批量处理待审内容。
- 永久删除、恢复已删除内容等高风险操作仅由超级管理员执行。
- 内容处置会保留审计记录；涉及证据保全或审核案件的内容需要走相应复核流程。

### 3.4 帖子作者溯源

对需要追查的帖子，使用“查看真实作者”：

1. 在内容管理中按标题或正文关键词定位帖子。
2. 打开帖子操作菜单，选择“查看真实作者”。
3. 确认后可查看 UID、用户名、校园邮箱和关联注册记录中的学号。

该功能不会提供发帖 IP。系统会先写入 `post.reveal_author` 审计记录，再读取身份和学号；关闭弹窗后，页面会清除已展示的身份信息。学号显示“未关联”表示未找到同邮箱的注册记录。

### 3.5 举报、审核案件与申诉

- 查看全部类型的举报及举报人身份。
- 处理统一审核案件，可作出放行、警告、隐藏、删除、禁言、封禁等决定。
- 对处罚申诉进行复核。
- 可在有必要时调阅案件作者身份。

### 3.6 注册、图片与聊天室

- 审核注册申请，并在必要时查看注册凭证截图。
- 查看、审核待处理图片；图片预览仅超级管理员可访问。
- 查看聊天室、被标记消息和消息记录，关闭聊天室或标记消息；涉及身份或发送环境的调阅会审计。

### 3.7 规则与系统功能

- 审核板块申请。
- 管理敏感词规则并重载词库。
- 发布和查看站内通知。
- 管理后台安全设置，例如二次验证。
- 使用私信溯源中心，以精确的用户、会话或消息 ID 查询私信参与者和发送环境。

### 3.8 审计日志

- 查看重要管理操作，并按操作管理员筛选。
- 审计日志保留处罚、权限变更、内容处置、规则变更、身份调阅和安全事件等重要行为。
- 打开用户列表、申诉列表、审计日志、敏感词列表等常规浏览行为不会新增审计记录；历史同类噪声记录不会在运营视图中显示。

## 4. 身份信息与对外协查

身份信息仅用于平台治理或具有授权依据的协查事项。建议操作顺序：

1. 先隐藏或保全涉事内容，并保留帖子 ID、链接、时间和内容证据。
2. 由超级管理员执行身份溯源；不要让普通管理员代为查询。
3. 在审计日志中确认调阅记录已经生成。
4. 按最小必要原则提交经授权的材料，不在无关群聊、截图或工单中传播 UID、邮箱、学号等信息。

## 5. 常见提示

| 提示 | 处理方式 |
| --- | --- |
| 无权访问 | 确认当前账号角色。普通管理员只能处理帖子举报。 |
| 举报已被其他管理员处理 | 刷新列表后查看最新状态。 |
| 学号显示“未关联” | 该账号没有匹配的注册记录；不要自行猜测或补填。 |
| 内容不能恢复 | 检查是否处于待审、证据保全或案件流程中，并在相应模块复核。 |
| 身份调阅失败 | 不应绕过失败重试获取数据；先检查审计存储和权限状态。 |
